# StarForth Experimental Figures (SVG)

This directory contains **standalone AND reusable** SVG figures generated from real experimental data.

## Why SVG?

SVG (Scalable Vector Graphics) files are:
- ✅ **Standalone**: Open directly in web browsers, no dependencies needed
- ✅ **Reusable**: Embed in LaTeX, HTML, Markdown, PowerPoint, etc.
- ✅ **Scalable**: Infinite resolution, perfect for publications
- ✅ **Editable**: Can be modified in Inkscape, Adobe Illustrator, or text editors
- ✅ **Small**: Typically smaller than equivalent PNG files

## Usage Examples

### 1. Standalone Viewing

Open any SVG file directly in a web browser:

```bash
firefox binary_2cycle_analysis.svg
# or
google-chrome heat_vs_jitter.svg
```

### 2. Embed in HTML

```html
<!-- Direct embedding -->
<img src="figures/tick_count_boxplot.svg" alt="Tick Count Analysis" width="800">

<!-- Or as an object for interactive features -->
<object data="figures/heat_vs_jitter.svg" type="image/svg+xml" width="720" height="432">
  Your browser does not support SVG
</object>
```

### 3. Include in LaTeX

#### Method A: Using `\includegraphics` (recommended)

```latex
\documentclass{article}
\usepackage{graphicx}

\begin{document}

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.8\textwidth]{figures/duration_vs_ticks.svg}
  \caption{Run Duration vs Tick Count}
  \label{fig:duration_ticks}
\end{figure}

\end{document}
```

**Note**: Requires `\usepackage{svg}` or shell-escape for SVG → PDF conversion:

```latex
\usepackage{svg}
\svgpath{{figures/}}
\includesvg[width=0.8\textwidth]{duration_vs_ticks}
```

#### Method B: Convert to PDF (one-time)

```bash
for svg in *.svg; do
  inkscape "$svg" --export-pdf="${svg%.svg}.pdf"
done
```

Then use standard `\includegraphics`:

```latex
\includegraphics[width=0.8\textwidth]{figures/duration_vs_ticks.pdf}
```

### 4. Include in Markdown

```markdown
![Tick Count Analysis](figures/tick_count_boxplot.svg)
```

### 5. Embed in PowerPoint/Google Slides

1. **Insert** → **Picture** → **From File**
2. Select the `.svg` file
3. Resize as needed (maintains perfect quality)

## Regenerating Figures

To regenerate SVG figures from experimental data:

```bash
cd experiments/l8_attractor_map
Rscript l8_analysis.R
```

Output SVGs will be in `results_*/analysis_output/charts/`.

## Technical Details

### SVG Specifications

- **Format**: SVG 1.1
- **Generated by**: R ggplot2 + ggsave(..., device="svg")
- **Fonts**: Embedded or system fonts (may vary by viewer)
- **Colors**: RGB color space
- **Compression**: Uncompressed XML (can be gzipped for web: `.svgz`)

### File Sizes

Total: ~1.2 MB for 11 figures (average ~110 KB per figure)

```bash
$ du -sh *.svg
192K    binary_2cycle_analysis.svg
 80K    duration_vs_ticks.svg
 72K    heat_timeseries_runNA.svg
 # ... etc
```

## Converting to Other Formats

### SVG → PNG (raster)

```bash
inkscape binary_2cycle_analysis.svg --export-png=output.png --export-dpi=300
```

### SVG → PDF (vector)

```bash
inkscape duration_vs_ticks.svg --export-pdf=output.pdf
```

### SVG → EPS (vector, for legacy LaTeX)

```bash
inkscape tick_count_boxplot.svg --export-eps=output.eps
```

## Viewing Tools

- **Web Browsers**: Chrome, Firefox, Safari, Edge (all support SVG natively)
