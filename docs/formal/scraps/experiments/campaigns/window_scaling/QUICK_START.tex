\section{James Law Validation: Quick-Start Guide}

\textbf{Purpose:} Validate James Law, $\Lambda = W / (\text{DoF} + 1)$, by
running the 2{,}880-run window-scaling experiment and confirming $K \approx 1.0$
across all conditions.

\subsection{Four-Step Workflow (Recommended)}

\subsubsection{Step 1: Setup and Validation ($\sim$5 minutes)}

\begin{lstlisting}[language=bash]
cd experiments/window_scaling_james_law
./setup_experiment.sh
\end{lstlisting}

Checks R, make, and gcc; generates the shuffled 2{,}880-row run matrix;
runs a test build and a test VM execution; estimates experiment duration.

\subsubsection{Step 2: Pre-Build All Configurations ($\sim$1.5 hours)}

\begin{lstlisting}[language=bash]
cd scripts/
./prebuild_all_configs.sh
\end{lstlisting}

Builds all 96 unique (DoF, Window) configurations once and stores them in
\texttt{experiments/bin/}. Binaries survive \texttt{make clean} and can be
reused for multiple experiment runs.

\subsubsection{Step 3: Execute the Experiment ($\sim$4--5 hours)}

\begin{lstlisting}[language=bash]
./run_window_sweep.sh

# Monitor progress in a second terminal:
tail -f ../results/raw/experiment.log
\end{lstlisting}

Executes 2{,}880 runs using pre-built binaries. No rebuilds: 3--5 hours
faster than the rebuild approach.

\subsubsection{Step 4: Analyze Results ($\sim$5 minutes)}

\begin{lstlisting}[language=bash]
cd scripts/
./analyze_results.R
\end{lstlisting}

Computes $K = \Lambda \times (\text{DoF}+1) / W$ for all runs, tests
$K \approx 1.0$, identifies the critical window threshold, and generates
five publication-quality plots.
