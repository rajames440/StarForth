\section*{Rule~1 Backlog Gate Implementation (November 2025)}

Rule~1 of the StarForth governance system defined four gates that any work
item must pass before entering the development backlog: template compliance,
QA approval, PM routing decision, and gatekeeper enforcement. This document
recorded the implementation of those gates in the GitHub Actions automation.
The governance system it describes is no longer in active use in the form
documented here. Retained for historical provenance.
