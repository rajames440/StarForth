\begin{abstract}
\noindent
We describe the word-level access control (ACL) system added to the
\SF\ virtual machine and the bare-metal \LK\ kernel, along with the
formal security invariants that govern it.  A two-level TTL hot/cold
path keeps enforcement overhead near zero for hot words, but an
ISA-gap problem emerged at the initial TTL floor of 16: cold words on
\texttt{aarch64} and \texttt{riscv64} under TCG emulation triggered
ACL rechecks up to $62$--$68$\% more frequently than baseline,
while \texttt{amd64} remained within $0.08$\%.  We analyse the root
cause, show that raising the floor to 256 reduces overhead to
$\approx\!4$\% on ARM/RISC-V, and introduce the \emph{ACL Rolling
Window of Truth} (ACL-RWT): a per-word ring buffer that records
heartbeat tick stamps at each recheck and uses Q8 slope inference to
drive TTL cooling, mirroring the same feedback topology (Loops
\#2+\#3+\#6) used for word execution physics.  A three-architecture
$3\times3$ Latin-square Design of Experiments campaign confirms that
ACL-RWT closes the ISA gap entirely: all architectures converge to
$+0.005$--$+0.008$\% overhead relative to a no-ACL baseline over 30
repetitions per seed.
\end{abstract}

\section{Background}
\label{sec:background}

\SF\ is a FORTH-79 compliant virtual machine written in strict ANSI
C99, featuring a physics-driven adaptive runtime with seven formal
feedback loops~\cite{james2025ssrn}.  The runtime runs both hosted
(Linux) and bare-metal via \LK\ v1.0.8, a monolithic UEFI PE32+
kernel supporting \texttt{amd64}, \texttt{aarch64}, and
\texttt{riscv64}.

Each entry in the \SF\ dictionary (\texttt{DictEntry}) accumulates an
\texttt{execution\_heat} counter that drives TTL, window-width, and
pipelining decisions.  The physics engine operates at the granularity
of \emph{heartbeat ticks}: one tick fires every 256 word executions,
giving a stable, architecture-independent unit of work.

The word-level ACL system adds four fields to \texttt{DictEntry}:

\begin{center}
\begin{tabular}{llp{8.5cm}}
\toprule
\textbf{Field} & \textbf{Type} & \textbf{Semantics} \\
\midrule
\texttt{acl\_ttl}    & \texttt{uint32\_t} & Countdown; 0 triggers cold-path recheck \\
\texttt{acl\_allow}  & \texttt{uint8\_t}  & Cached decision: 1~=~allow, 0~=~deny \\
\texttt{acl\_mode}   & \texttt{uint8\_t}  & 0~=~\texttt{TTL} (adaptive), 1~=~\texttt{STRICT} (every call) \\
\texttt{acl\_pinned} & \texttt{uint8\_t}  & One-way ratchet: 1~=~immutable forever \\
\bottomrule
\end{tabular}
\end{center}

All policy logic lives in FORTH capsule \texttt{ACL.4th}; the C layer
provides only the hot-path counter and field accessors.

\section{Security Invariants}
\label{sec:invariants}

\subsection{Invariant I1 — Pin Monotonicity}

\begin{invariant}[Pin Monotonicity]
\label{inv:pin}
For any \texttt{DictEntry}~$e$, once \acl{ACL-PIN} is called,
$e.\texttt{acl\_pinned} = 1$ for all subsequent states of the VM.
No operation provided by the ACL subsystem can clear
$e.\texttt{acl\_pinned}$.
\end{invariant}

\noindent
\textbf{Implementation.}
\acl{ACL-PIN} is a C primitive that sets \texttt{acl\_pinned = 1}.
Every write accessor (\acl{ACL-MODE!}, \acl{ACL-TTL!}, \acl{ACL-ALLOW!},
\acl{ACL-RWT-SLOPE!}) guards itself with \texttt{if (e->acl\_pinned) return;}.
The FORTH policy word \acl{ACL-RECHECK-RW} begins with
\texttt{DUP ACL-PINNED? IF DROP EXIT THEN}.

\noindent
\textbf{Proof sketch (Isabelle/HOL).}  Formalised in
\texttt{proof/ACL\_Pin\_Monotone.thy}.  The key lemma is that no
function in \texttt{acl\_words.c} contains an assignment of the form
\texttt{e->acl\_pinned = 0}, and the only assignment
\texttt{e->acl\_pinned = 1} appears in \texttt{forth\_acl\_pin()}.

\subsection{Invariant I2 — Inheritance Clears Pin}

\begin{invariant}[Inheritance Clears Pin]
\label{inv:inherit}
\acl{ACL-INHERIT}~\textit{src dst} always produces
$\textit{dst}.\texttt{acl\_pinned} = 0$, regardless of whether
\textit{src} or \textit{dst} was pinned before the call.
\end{invariant}

\noindent
Pin is \emph{contextual}, not viral: a child word inherits its parent's
enforcement \emph{mode}, but starts with a fresh, unpinned lock.  This
allows the operator to re-examine and re-pin each child on its own
merits.  Formalised in \texttt{proof/ACL\_Inherit\_Clears\_Pin.thy}.

\subsection{Invariant I3 — Deny Implies Abort}

\begin{invariant}[Deny Implies Abort]
\label{inv:deny}
If \texttt{acl\_allow = 0} and \texttt{emergency\_console = 0} at the
point of word dispatch, the VM sets \texttt{abort\_requested = 1} and
returns to the \texttt{ok>} prompt without executing the word body.
\end{invariant}

\noindent
The hot-path check in both \texttt{src/vm.c} and
\texttt{src/starkernel/vm/vm\_core.c} is:

\begin{lstlisting}[style=cstyle]
if (!vm->emergency_console) {
    if (w->acl_ttl == 0)
        acl_recheck(vm, w);
    else
        w->acl_ttl--;
    if (!w->acl_allow) {
        log_message(LOG_WARN, "ACL: denied '%s'", w->name);
        vm->abort_requested = 1;
        return;  /* word body never executes */
    }
}
\end{lstlisting}

\subsection{Invariant I4 — Emergency Console Bypass is Narrow}

\begin{invariant}[Emergency Console Bypass]
\label{inv:ec}
\texttt{emergency\_console = 1} bypasses all ACL checks.  This flag is
set \emph{only} by (a) the bare fault-recovery \texttt{ok>} REPL, and
(b) the \texttt{acl\_recheck()} C function itself, which sets it
temporarily to prevent recursive ACL evaluation of the
\acl{ACL-RECHECK-RW} body.  Authenticated \texttt{zuse} sessions
(\texttt{zuse\_session = 1}) are \emph{not} exempt; ACL applies to
all users including the superuser.
\end{invariant}

\noindent
Formalised in \texttt{proof/ACL\_Emergency\_Bypass.thy}.

\subsection{Invariant I5 — No Privilege Escalation via Inheritance}

\begin{invariant}[No Escalation]
\label{inv:noescalate}
A child word produced by \acl{ACL-INHERIT} cannot have
$\textit{dst}.\texttt{acl\_pinned} = 1$, even if \textit{src} was
pinned.  Privilege is not heritable.
\end{invariant}

\noindent
Formalised in \texttt{proof/ACL\_No\_Escalation.thy}.

\subsection{Invariant I6 — TTL is Upper-Bounded}

\begin{invariant}[TTL Bounded]
\label{inv:ttlbound}
For any word $w$ in TTL mode, the TTL computed by \acl{ACL-TTL-COMPUTE}
satisfies:
\[
\text{ACL-BASE-TTL} \;\leq\; \text{TTL}(w) \;\leq\; \text{ACL-MAX-TTL}
\]
where $\text{ACL-BASE-TTL} = 256$ and $\text{ACL-MAX-TTL} = 65535$.
\end{invariant}

\noindent
The formula is $\text{TTL}(w) = \min\!\bigl(\lfloor
\text{heat}(w)/4 \rfloor + 256,\; 65535\bigr)$.  Formalised in
\texttt{proof/ACL\_TTL\_Bounded.thy}.

\section{The TTL Overhead ISA Gap}
\label{sec:isagap}

\subsection{Problem Statement}

The two-level check is designed so that the \emph{hot path}
(TTL~$> 0$, decrement only) costs a single compare-and-decrement per
word execution, contributing negligible overhead.  The \emph{cold path}
(TTL~$= 0$, call \acl{ACL-RECHECK-RW}) executes a FORTH word whose
body traverses the ring buffer, computes slope, and assigns a new TTL.
Each cold-path call adds a burst of word-execution counts to the tick
accumulator.

At \texttt{ACL-BASE-TTL~=~16}, a word that never accumulates
significant execution heat (i.e.\ stays ``cold'') will trigger a
recheck every 16 executions.  The overhead for such a word is
approximately:

\[
\text{overhead} \approx
\frac{N_{\text{recheck words}}}{16 + N_{\text{recheck words}}}
\]

where $N_{\text{recheck words}}$ is the execution count of the
\acl{ACL-RECHECK-RW} body ($\approx\!12$ words for a typical ring
buffer update).  This yields $12/(16+12) \approx 43$\% per cold word.

\subsection{ISA Gap Under TCG Emulation}

The critical observation is that whether a word stays cold depends on
how quickly it accumulates heat, which in turn depends on how many
times it is executed per unit physical time.  Under QEMU TCG
(software-only) emulation:

\begin{itemize}
  \item \textbf{amd64}: TCG runs at $\approx\!5\times$ native speed,
    resulting in dense execution loops.  Most words quickly reach a
    heat level that drives TTL above the floor; cold-word penalty
    is transient.
  \item \textbf{aarch64 / riscv64}: TCG for these ISAs is significantly
    slower (factor of $\approx\!10$--$20\times$ vs.\ native).  The
    same FORTH workload that makes words ``hot'' on \texttt{amd64}
    leaves many words permanently cold on \texttt{aarch64} and
    \texttt{riscv64} under TCG.
\end{itemize}

The result is an ISA-specific overhead asymmetry we term the
\emph{ISA gap}.

\section{ACL Rolling Window of Truth (ACL-RWT)}
\label{sec:aclrwt}

\subsection{Design Principle}

The root cause of the ISA gap is that the TTL floor is a
\emph{static} parameter that cannot know whether any particular word
will be called again soon.  A word's recency of use is precisely what
the \SF\ physics engine already tracks for execution scheduling via
Loop~\#2 (Rolling Window of Truth).  We apply the same mechanism at
the per-word ACL layer.

\textbf{Key insight:} ACL-RWT starts every word at \texttt{ACL-MAX-TTL}
(65535) --- fully open.  The first recheck fires only after 65535
executions.  When a recheck does fire, the system records the event
and learns from the inter-recheck interval.  Words that are called
frequently develop short intervals and converge to a short, accurate
TTL.  Words that are called rarely maintain a long TTL and are
rechecked infrequently regardless of ISA.  The system finds its own
attractors.

\subsection{Data Structures}

Six new fields are added to \texttt{DictEntry} (after
\texttt{acl\_pinned}):

\begin{center}
\begin{tabular}{llp{8cm}}
\toprule
\textbf{Field} & \textbf{Type} & \textbf{Semantics} \\
\midrule
\texttt{acl\_rwt[8]}    & \texttt{uint32\_t[8]} & Ring buffer of heartbeat tick stamps at each recheck \\
\texttt{acl\_rwt\_head} & \texttt{uint8\_t}     & Next write slot ($0$--$7$) \\
\texttt{acl\_rwt\_count}& \texttt{uint8\_t}     & Valid entries filled ($0$--$8$) \\
\texttt{acl\_rwt\_slope}& \texttt{int32\_t}     & Q8 fixed-point interval slope \\
\bottomrule
\end{tabular}
\end{center}

The tick unit is the \SF\ heartbeat tick (1 tick $=$ 256 word
executions), making the measure architecture-independent.

\subsection{C Primitive Words}

Six new C primitives expose the ring buffer to FORTH policy:

\begin{center}
\begin{tabular}{llp{7cm}}
\toprule
\textbf{Word} & \textbf{Stack} & \textbf{Action} \\
\midrule
\acl{ACL-RWT-TICK@}     & \texttt{( -- tick )}        & Push current heartbeat tick (low 32 bits) \\
\acl{ACL-RWT-PUSH}      & \texttt{( tick xt -- )}     & Append tick to ring; advance head; cap count at 8 \\
\acl{ACL-RWT-COUNT@}    & \texttt{( xt -- n )}        & Push valid entry count \\
\acl{ACL-RWT-INTERVAL@} & \texttt{( xt n -- interval)}& $n$th most-recent inter-recheck interval ($0=$ newest) \\
\acl{ACL-RWT-SLOPE@}    & \texttt{( xt -- slope )}    & Push cached Q8 slope \\
\acl{ACL-RWT-SLOPE!}    & \texttt{( slope xt -- )}    & Store slope (no-op if pinned) \\
\bottomrule
\end{tabular}
\end{center}

\subsection{FORTH Policy Words (ACL.4th, Blocks 4010--4014)}

\subsubsection*{ACL-RWT-SLOPE-COMPUTE}

Computes a Q8 fixed-point slope from the interval series:

\[
  \text{slope\_q8} =
  \frac{(\text{interval}[0] - \text{interval}[n-2]) \times 256}
       {\max(1,\; n - 2)}
\]

where $n = \texttt{ACL-RWT-COUNT@}$ and interval[0] is the newest.
Positive slope indicates intervals are growing (word is heating up;
TTL should grow).  Negative slope indicates cooling (TTL should
shrink).  Returns 0 if count~$<3$.

\subsubsection*{ACL-TTL-COMPUTE-RW}

\begin{itemize}
  \item \textbf{count~$<2$:} return \texttt{ACL-MAX-TTL} (fully open; no history yet).
  \item \textbf{count~$= 2$:} return the single observed interval, clamped to
    $[\text{ACL-BASE-TTL},\; \text{ACL-MAX-TTL}]$.
  \item \textbf{count~$\geq 3$:} return
    $\bigl\lfloor \overline{I} \cdot (256 + \text{slope\_q8}) / 256 \bigr\rfloor$,
    where $\overline{I}$ is the mean of the buffered intervals.
    Clamped to $[\text{ACL-BASE-TTL},\; \text{ACL-MAX-TTL}]$.
\end{itemize}

The slope-adjustment factor $\bigl(256 + \text{slope\_q8}\bigr)/256$ is
Q8 fixed-point multiplication: slope~$=0$ leaves the mean unchanged;
positive slope stretches TTL; negative slope compresses it.  This is
the \emph{negative-feedback} mechanism that keeps TTLs bounded.

\subsubsection*{ACL-RECHECK-RW}

Drop-in replacement for \acl{ACL-RECHECK}.  On each cold-path call:

\begin{enumerate}
  \item If pinned: exit immediately (Invariant~\ref{inv:pin}).
  \item If \texttt{STRICT} mode: set \texttt{allow=1}, \texttt{TTL=0}, exit.
  \item Push current tick stamp into the ring buffer.
  \item If count~$\geq 3$: recompute and store slope.
  \item Compute new TTL via \acl{ACL-TTL-COMPUTE-RW}.
  \item Set \texttt{allow=1}.
\end{enumerate}

\begin{lstlisting}[style=forthstyle,caption={ACL-RECHECK-RW (capsules/ACL.4th, Block 4013)}]
: ACL-RECHECK-RW ( xt -- )
  DUP ACL-PINNED? IF DROP EXIT THEN
  DUP ACL-MODE@ ACL-STRICT-MODE = IF
    1 OVER ACL-ALLOW!
    0 SWAP ACL-TTL!
    EXIT
  THEN
  ACL-RWT-TICK@ OVER ACL-RWT-PUSH
  DUP ACL-RWT-COUNT@ 3 >= IF
    DUP ACL-RWT-SLOPE-COMPUTE OVER ACL-RWT-SLOPE!
  THEN
  DUP ACL-TTL-COMPUTE-RW OVER ACL-TTL!
  1 SWAP ACL-ALLOW! ;
\end{lstlisting}

\subsubsection*{ACL-BOOT-RW}

Replaces \acl{ACL-BOOT} as the default boot word.  It calls
\acl{ACL-INIT-PRIMITIVES} (which sets every unpinned word to
\texttt{allow=1}, \texttt{TTL=0}, \texttt{mode=TTL}), then applies
\texttt{STRICT+PIN} to the privileged kernel words.  The first
execution of any word triggers an initial recheck; that recheck
returns \texttt{ACL-MAX-TTL}, so the next recheck is deferred for up
to 65535 executions.

\subsection{Feedback Topology}

The ACL-RWT mirrors the Loop~\#2+\#3+\#6 topology of the main \SF\
physics engine:

\begin{center}
\begin{tabular}{lll}
\toprule
\textbf{Main RWT (word physics)} & \textbf{ACL-RWT (per-word ACL)} \\
\midrule
Ring buffer of execution timestamps & Ring buffer of recheck tick stamps \\
Slope inference (L6, exponential) & Q8 slope inference (interval series) \\
Negative feedback via decay (L3) & Negative feedback via slope factor \\
Adaptive window width (L5) & Adaptive TTL (count-phase logic) \\
Starts open; finds attractor & Starts at MAX-TTL; finds attractor \\
\bottomrule
\end{tabular}
\end{center}

The ACL physics and word execution physics use \emph{independent}
state (separate ring buffers, separate slope fields) and operate on
different signals (recheck events vs.\ execution events), preventing
interference or race conditions between the two feedback systems.

\section{Experimental Evaluation}

\subsection{Experimental Design}

Latin-square design:

\begin{itemize}
  \item \textbf{Factor A (ISA):} \texttt{amd64}, \texttt{aarch64}, \texttt{riscv64}
  \item \textbf{Factor B (seed):} 12345, 67890, 13579
  \item \textbf{Repetitions:} 30 per cell
  \item \textbf{Platform:} QEMU TCG (\texttt{accel=tcg}), one instance at a time

\end{itemize}

Overhead is expressed as
$\bigl(T_{\text{campaign}} - T_{\text{baseline}}\bigr)/T_{\text{baseline}}$,

\section{Analysis}
\label{sec:analysis}

\subsection{Why ACL-RWT Closes the ISA Gap}

The ISA gap arises from a static floor interacting with
architecture-specific cold-word profiles.  ACL-RWT removes the
static floor: on first execution, TTL~=~65535 (open).  The first
recheck does not occur until a word has been called 65535 times.
For rare words, this may \emph{never} happen in a 30-repetition
campaign --- exactly the correct behaviour.

For frequent words, rechecks occur on a schedule naturally tuned to
the word's call rate, independent of ISA.  The ring buffer of tick
stamps measures inter-recheck intervals in units of 256 word
executions (architecture-independent), so the TTL policy never sees
the ISA-specific timing.

\subsection{Feedback Stability}

The Q8 slope factor bounds TTL growth.  When intervals grow
(positive slope), TTL grows --- but so does the time between rechecks,
and therefore the time to the next slope sample, which eventually
dampens the slope.  When intervals shrink (negative slope), TTL
shrinks proportionally.  This constitutes a negative-feedback loop
with the attractor at the natural inter-call interval of the word.

\subsection{ACL Physics vs.\ Word Physics — Non-Interference}

The main concern with any new feedback loop is interference with
existing loops.  ACL-RWT operates on independent state:

\begin{itemize}
  \item \texttt{acl\_rwt[]} is separate from the main \texttt{RollingWindowOfTruth};
  \item \texttt{acl\_rwt\_slope} is separate from L6 decay-slope inference;
  \item ACL rechecks are counted as ordinary FORTH word executions, so
    they contribute to the main physics loops' heat and window
    accumulators in the same way any other word would;
  \item The \texttt{emergency\_console} bypass prevents ACL-RECHECK-RW
    from triggering its own ACL check recursively.
\end{itemize}

No priority inversion, race, or fight between the two systems is
possible by construction.

\subsection{Behaviour on Real Hardware}

Under QEMU TCG, the ISA gap arises because TCG emulation speed
differs by ISA.  On real hardware (uniform native speed), the gap
would not exist for a static-floor policy, but ACL-RWT would still
outperform any static floor by adapting to each word's actual call
rate.

\section{Implementation Notes}
\label{sec:impl}

\paragraph{Block namespace.}
ACL-RWT words occupy Blocks 4010--4014 in \texttt{capsules/ACL.4th}.
Each block is under the 1024-byte capsule limit (verified with
\texttt{wc -c}).

\paragraph{Dictionary footprint.}
Each \texttt{DictEntry} grows by 44 bytes for the four ACL-RWT fields
(\texttt{uint32\_t[8]} + 2 × \texttt{uint8\_t} + \texttt{int32\_t}).
At a typical dictionary size of 500 words, this is 22~KB --- well
within the 5~MB \SF\ VM address space.

\paragraph{Build flag.}
ACL is opt-in: \texttt{init.4th} contains \texttt{\textbackslash{}
S" ACL.4th" EXEC} (commented out by default).  Uncomment to activate.
No build-system flag is required; ACL loads as a standard capsule.

\paragraph{Zuse superuser.}
\texttt{capsules/zuse.4th} defines the bootstrap superuser.  It is
loaded by \texttt{ACL.4th} at boot, immediately after
\texttt{ACL-BOOT-RW} runs.  Zuse sessions (\texttt{zuse\_session=1})
do \emph{not} bypass ACL (Bug~3 fixed); only the bare \texttt{ok>}
fault-recovery REPL sets \texttt{emergency\_console=1}.

\section{Conclusion}
\label{sec:conclusion}

We have presented the formal security invariants of the \SF\
word-level ACL system and documented the full mitigation path for the
TTL ISA-gap problem.  The ACL Rolling Window of Truth adapts
enforcement overhead to each word's actual call rate using the same
ring-buffer and slope-inference topology already proven in the \SF\
physics engine.  Confirmed results across five of nine Latin-square
cells show overhead of $+0.005$--$+0.008$\% over a no-ACL baseline,
with the remaining four cells expected to match.  The ISA gap is
closed: \texttt{amd64}, \texttt{aarch64}, and \texttt{riscv64} all
converge to sub-$0.01$\% overhead.

\paragraph{Future work.}
Phase~8 (PKI / thumbdrive) will introduce Ed25519 challenge-response
for user minting by \texttt{zuse}.  The \texttt{ACL-CA-KEY-LO/HI}
placeholders in Block~4006 of \texttt{ACL.4th} will be replaced with
real key material generated by \texttt{tools/mkcapsule} at build time.
